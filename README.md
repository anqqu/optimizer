# 🚀 XanMod + BBr + TFO + MSS clamp Optimizer для КВН

![XanMod](https://img.shields.io/badge/XanMod-Latest-blue)
![BBRv3](https://img.shields.io/badge/BBRv3-Enabled-brightgreen)
![OS](https://img.shields.io/badge/OS-Debian%20%7C%20Ubuntu-blue)

**Мощная автоматическая оптимизация сетевого стека и ядра сервера под КВН (подходит для конфигураций 1/1, 1/2, 2/4 и выше).**

**Для конфигурации 1/1 требуется Swap-файл.**

---

### ✨ Что делает скрипт


| Компонент | Статус | Описание |
| :--- | :---: | :--- |
| **XanMod Kernel** | 🤖 Авто | Автоматический выбор версии (v2/v3/v4) под архитектуру вашего CPU |
| **BBRv3** | ✅ Включён | Современный алгоритм управления перегрузкой TCP для снижения потерь |
| **TCP Fast Open (TFO)** | ✅ Включён | Значительно ускоряет установку повторных TCP-соединений |
| **MSS Clamping** | ✅ Включён | Оптимизация размера пакетов через nftables (IPv4 + IPv6) |
| **Conntrack** | 📈 Увеличен | Расширение лимита до 524 288 одновременных соединений |
| **CAKE qdisc** | 🔄 Заменён | Отключается в пользу более производительной связки `mq` + `fq` |
| **Автозапуск** | ✅ Включён | Все тюнинг-настройки sysctl автоматически применяются после ребута |

---

### ⚠️ Важное предупреждение

> [!IMPORTANT]
> **Скрипт заменяет ядро системы.** Используйте его только на серверах, где вы готовы к перезагрузке и кратковременному даунтайму.
> * Перед запуском обязательно сделайте **снапшот (snapshot)** или бэкап VPS.
> * Убедитесь, что у вас останется альтернативный доступ к серверу (например, через VNC в панели хостинга).

---

### 📋 Требования и рекомендации

* **Идеальные условия:** Чистая ОС (Fresh install) на VPS с виртуализацией **KVM**.
* **Ресурсы:** Рекомендуется минимум **2 ядра CPU и 2 ГБ RAM**. 
  * *Примечание:* На конфигурациях с 1 CPU и 1-2 ГБ RAM скрипт также работает хорошо, но для высоких сетевых нагрузок 2 ГБ и более оперативной памяти являются наиболее стабильным вариантом.
* **Поддерживаемые ОС:** 
  * `Debian 11 / 12`
  * `Ubuntu 22.04 / 24.04 LTS`
* **Ограничения:** Скрипт не предназначен для работы на виртуализации **OpenVZ / LXC**, так как на них невозможно изменить параметры ядра системы (`sysctl`) и обновить само ядро до XanMod.

### 🚀 Установка и запуск

#### Вариант 1: Автоматическая установка (одной командой)
```bash
sudo curl -fsSL https://raw.githubusercontent.com/anqqu/optimizer/main/optimizer.sh | sudo bash
```

#### Вариант 2: Скачивание и запуск вручную
```bash
sudo curl -fsSL -O https://raw.githubusercontent.com/anqqu/optimizer/main/optimizer.sh
sudo chmod +x optimizer.sh
sudo ./optimizer.sh
```

### 🔄 Завершение и проверка

После успешного выполнения скрипта **обязательно** перезагрузите сервер:
```bash
sudo reboot
```

После перезагрузки проверьте корректность работы всех компонентов:
```bash
# Проверка версии ядра XanMod
sudo uname -r
6.1.x-xanmod1
# главное — наличие флага -xanmod

# Проверка активного алгоритма TCP
sudo sysctl net.ipv4.tcp_congestion_control
# Должно быть net.ipv4.tcp_congestion_control = bbr

# 3. Проверка очереди пакетов qdisc (ищите связку "mq" и "fq" для сетевых интерфейсов)
tc qdisc show
# Ожидаемый ответ:
# qdisc mq 0: dev eth0 root               <-- Указывает на многопоточную очередь сетевой карты
# qdisc fq 0: dev eth0 parent :1 limit... <-- Указывает на активный алгоритм Fair Queueing для каждого ядра

# 4. Проверка правил nftables (убедитесь, что брандмауэр активен и защищает порты)
sudo nft list ruleset
# Ожидаемый ответ: структура правил inet filter, где по умолчанию входящий трафик сбрасывается (policy drop),
# но явно разрешены порты вашего SSH (22) и Xray (обычно 443).
```

### 🛠️ Бонус: Быстрое создание 1GB Swap

Если вашему серверу не хватает оперативной памяти, выполните эту команду. Она безопасно настроит swap-файл с использованием `sudo` для каждого шага:

```bash
sudo swapoff /swapfile 2>/dev/null || true && \
sudo fallocate -l 1G /swapfile && \
sudo chmod 600 /swapfile && \
sudo mkswap /swapfile && \
sudo swapon /swapfile && \
{ sudo grep -qE '^/swapfile\s' /etc/fstab || echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab; }
```

<details>
<summary><b>🔍 Пошаговый разбор работы команды Swap (нажмите, чтобы развернуть)</b></summary>



| Команда | Что она делает |
| :--- | :--- |
| `sudo swapoff /swapfile 2>/dev/null \|\| true` | Деактивирует старый swap-файл, если он существовал ранее. |
| `sudo fallocate -l 1G /swapfile` | Мгновенно резервирует 1 ГБ места на диске под новый файл. |
| `sudo chmod 600 /swapfile` | Задает максимальные права безопасности (чтение/запись только для `root`). |
| `sudo mkswap /swapfile` | Форматирует созданный файл в системную структуру swap. |
| `sudo swapon /swapfile` | Включает подкачку в текущей сессии без необходимости перезагрузки. |
| `sudo grep -qE ... `| Проверяет `/etc/fstab` на наличие swap и безопасно записывает автозапуск в конец файла. |

</details>
